# 調教SP 仕様書

## 概要

調教SP（Special Performance）は、出走馬の調教状態を数値化し、A〜Eの5段階グレードで表示する独自指数です。
回収率が段階的に向上するよう設計されており、**グレードが高いほど単勝回収率が高い**という単調性を保証しています。

## スコア算出式

### STEP 1：ランクボーナス（0〜+6点）

同じレースに出走する馬の中で「追切指数（T_CYB.oi_index）」「仕上指数（T_CYB.shiage_index）」それぞれについて降順ランクを算出し、以下のボーナスを与える。

| 順位 | ボーナス |
|---|---|
| 1位 | +3 |
| 2位 | +2 |
| 3位 | +1 |
| 4位以下 | 0 |

**ランクボーナス = oi順位ボーナス + shiage順位ボーナス（最大+6）**

同値タイは同順位（SQLのRANK()相当）。追切指数または仕上指数のどちらかが0（欠損）の場合は調教SP算出不可（表示なし）。

### STEP 2：z-scoreネガティブ調整（≤0点）

レース内での相対的な低さをペナルティとして反映する。レース内平均・標準偏差を使ったz-scoreを計算し、**平均を下回る場合のみ**減点する。

```
oi_z  = (oi_index  - レース内平均oi)  / レース内標準偏差oi
shi_z = (shiage_index - レース内平均shi) / レース内標準偏差shi

z調整 = MIN(0, ROUND((oi_z + shi_z) × 1.0))
```

平均以上（z_combined ≥ 0）の場合は調整なし（0点）。
これにより「弱い集団の中の1位」（絶対値は低いが順位は高い馬）を適切にペナルティする。

### STEP 3：調教矢印ペナルティ（T_KYI.chokyo_yajirushi）

| コード | 矢印 | 点数 |
|---|---|---|
| 1 | ↑↑ | 0（加点なし） |
| 2 | ↑  | 0（加点なし） |
| 3 | →  | 0 |
| 4 | ↓  | **−2** |
| 5 | ↓↓ | **−4** |

> **設計上の注意**：↑↑や↑は加点しない。
> 理由：市場に広く認知されている上昇サインを加点すると「バレやすい馬」として回収率が下がることをデータで確認した。ペナルティのみで非対称に運用する。

### STEP 4：放牧先ランクペナルティ（T_KYI.hohbokusaki_rank）

| ランク | 点数 |
|---|---|
| A / B / C | 0 |
| D | **−2** |
| E | **−4** |

### 合計スコア

```
調教SPスコア = ランクボーナス + z調整 + 矢印ペナルティ + 放牧先ペナルティ
```

| 理論最大値 | +6（追切・仕上ともに1位、ペナルティなし） |
| 理論最小値 | 制限なし（約−14程度が実務上の下限） |

## 実装

### フロントエンド（public/entries.html）

`calcChokyoSP(rows)` 関数がレース全馬のデータを受け取り、各馬のグレードを返す。
APIからは既存の `oi_index`、`shiage_index`（T_CYB）、`chokyo_yajirushi`、`hohbokusaki_rank`（T_KYI）を利用するため、バックエンド変更は不要。

### バックエンド変更なし

`/api/races/:raceKey/entries` の既存レスポンスに必要なフィールドがすべて含まれている。

## 開発経緯

1. **v0（原案）**：追切・仕上のレース内順位ボーナス + 調教印 + 固定閾値ペナルティ（両方<50）
2. **v1**：矢印・放牧先ランクを追加、固定閾値ペナルティを維持
3. **z-scoreアプローチ**：固定閾値をz-scoreに置き換え（連続値化）
4. **ハイブリッド**：プラス側はランク方式、マイナス側はz-scoreを採用
5. **A案（採用）**：カテゴリ変数のポジティブボーナスを撤廃、ペナルティのみに統一 → グレードバンド制で単調性を確保

### 単調性が崩れた原因と対策

- **「両方1位」問題**：追切・仕上ともにレース1位の馬はスコア最高値になるが、市場に丸見えのため過剰に投票され回収率が下がる（いわゆる「バレやすい」現象）。
- **対策**：グレードバンド制を採用。整数スコアレベルではなく5段階に集約することで完全な単調性を実現。
- **カテゴリ変数の扱い**：↑↑矢印や◎調教印を加点すると、同じ現象（バレやすさ）が再現されることをグリッドサーチで確認。ペナルティのみに限定した。
