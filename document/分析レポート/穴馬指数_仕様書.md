# EX指数 仕様書

> 最終更新: 2026-05-17  
> ※ 旧称「穴馬指数」。2026-05-17 に EX指数（Explosion Index）に改称。内部テーブル名・フィールド名（`anaba_*`）は変更なし。

---

## 1. 概要

**EX指数**は、JRDBのあらゆるファクターを組み合わせて「この馬が激走する期待値はどのくらいか」を数値化した指数です。

出馬表の `EX指数 全体` 列と `EX指数 コース` 列に表示され、馬券戦略の補助情報として使います。

---

## 2. 対象馬の定義

| 項目 | 定義 |
|------|------|
| ETL対象 | `kijun_odds >= 10.0` の馬のみ `T_ANABA_RACE_LOG` に格納 |
| スコア計算 | **出走全馬**（T_ANABA_SCORE）に対して算出 |
| 表示時の背景色 | `kijun_odds >= 10.0` の馬のみ背景色を付ける |
| 非対象ETL | 障害レース（`tds_code='3'`）、新馬戦（`class='A1'`） |

> バックテストの結果、EX指数は10倍未満の馬に対して識別力を持たないため（回収率に有意な相関なし）、背景色によるハイライトは穴馬のみに限定している。

---

## 3. 指数の仕組み

### 3.1 基本思想

過去データから「このファクター値を持つ穴馬は平均よりどれだけ多く回収できるか」を測り、その偏差を合算してスコアとする。

```
指数スコア = Σ（各ファクターの単勝回収率 − ベースライン単勝回収率）
```

- **ベースライン**: 全穴馬（kijun_odds ≥ 10）の平均単勝回収率（約69%）
- **各ファクタースコア**: そのファクター値を持つ穴馬の回収率とベースラインの差（単位: %ポイント）

### 3.2 指数の種類

| 指数 | 説明 | APIフィールド |
|------|------|-------------|
| EX指数（全体） | 全競馬場・全距離のデータで算出 | `anaba_overall_score` |
| EX指数（コース別） | 場コード×芝ダ×距離帯で補正 | `anaba_course_score` |

### 5.3 全体指数 vs コース別指数

- **全体指数 > コース別指数**: そのコースでは展開・脚質が通常ほど有利でない
- **コース別指数 > 全体指数**: そのコースに特に向いたファクター構成

コース別データが少ない場合（HAVING COUNT(*) < 20）はフォールバックして全体値を使用。

## 6. テーブル構成

### 6.1 T_ANABA_RACE_LOG（ファクトテーブル）

```sql
-- kijun_odds >= 10.0 の馬のみ格納
PRIMARY KEY (course_code, year_code, kai, day_code, race_num, uma_num)
```

| カラム | 型 | 説明 |
|--------|-----|------|
| `finish_order` | CHAR(2) | 着順（T_SED） |
| `win_payout` | INT | 単勝払戻（円） |
| `place_payout` | INT | 複勝払戻（円） |
| `is_dual_top` | TINYINT | テン≤3 AND 上がり≤3 |
| `is_sen_oki` | TINYINT | テン≥7 AND 上がり≤2 |
| `is_hana_iki` | TINYINT | テン≤2 AND 上がり≥7 |
| `is_mid_chaser` | TINYINT | 位置3〜5 AND 上がり≤3 |

### 6.2 T_ANABA_FACTOR_AGG（ファクター別集計、EAV形式）

```sql
PRIMARY KEY (factor_type, factor_value, course_code, tds_code, dist_band)
INDEX idx_kyusha_anaba_covering (factor_type, factor_value, trainer_code, total_count, place_payout_sum)
```

### 6.3 T_ANABA_SCORE（指数テーブル、出馬表表示用）

```sql
PRIMARY KEY (course_code, year_code, kai, day_code, race_num, uma_num)
```

| カラム | 説明 |
|--------|------|
| `overall_score` | EX指数（全体）DECIMAL(7,1) |
| `course_score` | EX指数（コース別）DECIMAL(7,1) |
| `score_ten`〜`score_blood` | 各ファクターの寄与値（内訳） |

> T_ANABA_SCORE は **出走全馬** を対象に計算する。

## 7. ETLフロー

`sql/anaba_index.sql` の4パート構成：

```
Part 1: CREATE TABLE IF NOT EXISTS（T_ANABA_RACE_LOG, T_ANABA_FACTOR_AGG, T_ANABA_SCORE）
Part 2: INSERT INTO T_ANABA_RACE_LOG（kijun_odds>=10 の馬のみ）
Part 3: INSERT INTO T_ANABA_FACTOR_AGG（14ファクター × 全体+コース別）
Part 4: INSERT INTO T_ANABA_SCORE（全馬対象・約3〜5分）
```

### 実行方法

`download.html` の「EX指数を更新する」ボタン、またはCLI：

```powershell
mysql -h localhost -P 3306 -u root -p racing < sql/anaba_index.sql
```

### 実行タイミング

| タイミング | 理由 |
|-----------|------|
| SED取込後 | win_payout / finish_order が更新され集計精度向上 |
| 週次（土日前） | 最新KYIデータで指数を最新化 |

## 8. API

### POST /api/anaba-etl

EX指数 ETL を実行。SSEで進捗を返す。

### GET /api/races/:raceKey/entries

T_ANABA_SCORE を LEFT JOIN し `anaba_overall_score`、`anaba_course_score`、`score_*`（内訳13種）を返却。
