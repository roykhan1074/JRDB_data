# 展開適合指数 仕様書

最終更新: 2026-05-24

---

## 1. 概要

**展開適合指数（Pace Fit Index）** は、JRDB のペース予想・テン指数順位・上がり指数順位を組み合わせ、各馬が「その展開にどれだけ適合しているか」を数値化した指数。

スコアはベースラインとの単勝回収率の差を積み上げた偏差合計として表現される。正の値ほどプラス材料、負の値ほどマイナス材料が多いことを意味する。

---

## 2. 対象レース

| 条件 | 内容 |
|------|------|
| 芝・ダート | `T_BAC.tds_code IN ('1','2')` |
| 障害 | **除外**（`tds_code='3'`） |
| ペース予想 | `T_KYI.pace_yoso IN ('H','M','S')` の馬のみスコアあり |

カバレッジ: 芝・ダート出走馬の **99.98%** にスコアが付与される（pace_yoso が未記入の馬のみ欠損）。

---

## 3. 使用 JRDB フィールド

| テーブル | フィールド | 用途 |
|---------|-----------|------|
| `T_KYI` | `pace_yoso` | ペース予想（H=ハイ / M=ミドル / S=スロー） |
| `T_KYI` | `ten_index_juni` | テン指数順位（レース内の前半速度ランク） |
| `T_KYI` | `ten_index` | テン指数（ギャップ計算に使用） |
| `T_KYI` | `agari_index_juni` | 上がり指数順位（レース内の末脚ランク） |
| `T_KYI` | `agari_index` | 上がり指数（ギャップ計算に使用） |
| `T_BAC` | `tds_code` | 芝ダ判定（対象レース絞り込み） |
| `T_BAC` | `course_code` / `distance` | コース補正（course_score 計算用） |
| `T_SED` | `win` / `place` | 単勝・複勝払戻金額（ファクター回収率集計用） |

## 4. スコア計算ロジック（overall_score）

出馬表に表示されるのは **overall_score**（コース補正なし）のみ。

### 4-1. ファクター回収率の事前集計

以下の組み合わせを `T_PACEFIT_FACTOR_AGG` に集計する（HAVING ≥ 30件のみ有効）。

| factor_type | キー（factor_value） |
|-------------|---------------------|
| `baseline` | `''`（全体平均） |
| `pace_ten` | `"{pace_yoso}\|{ten_band}"` |
| `pace_agari` | `"{pace_yoso}\|{agari_band}"` |
| `gap_ten_h` | `'H'`（H pace × テン1位 × ギャップ大） |
| `gap_agari_m` | `'M'`（M pace × 上がり1位 × ギャップ大） |

順位帯の区切り: `1 / 2 / 3-4 / 5-8 / 9+`

### 4-2. overall_score の計算式

```
overall_score = score_ten_o + score_agari_o + score_gap
```

| 項目 | 計算式 |
|------|--------|
| `score_ten_o` | `pace_ten[pace×ten_band] の回収率 − baseline 回収率` |
| `score_agari_o` | `pace_agari[pace×agari_band] の回収率 − baseline 回収率` |
| `score_gap` | ギャップボーナス（下記参照）、なければ 0 |

ファクター集計が 30 件未満でレコードなしの場合は baseline 値で代替（偏差 = 0）。

### 4-3. ギャップボーナス

| 条件 | ボーナスの計算 |
|------|--------------|
| `pace_yoso='H'` かつ `ten_index_juni=1` かつ `ten_index(1位) − ten_index(2位) ≥ 2.0` | `gap_ten_h の回収率 − pace_ten[H\|1] の回収率` |
| `pace_yoso='M'` かつ `agari_index_juni=1` かつ `agari_index(1位) − agari_index(2位) ≥ 2.0` | `gap_agari_m の回収率 − pace_agari[M\|1] の回収率` |

ハイペース × テン最速が突出している馬、ミドルペース × 末脚最速が突出している馬に追加加点する設計。

---

## 5. course_score（コース補正）について

コース × 芝ダ × 距離帯（短〜1400 / マイル / 中距離 / 長距離）でセルごとに集計したスコア。

**検証結果：過学習が確認されたため、出馬表には表示しない。** overall_score のみを使用する。

> 30+帯: in-sample 129.8% → OOS 67.3%（overall_score は 105.1% → 91.6% と安定）

## 7. 出馬表での色分け

| スコア | 背景色 | 意味 |
|--------|--------|------|
| **25以上** | `#fbcfe8`（ピンク） | 展開が大きくプラス |
| **15〜24** | `#fef9c3`（薄黄） | 展開がプラス |
| **0〜14** | 色なし | ニュートラル |
| **負** | `#e5e7eb`（グレー） | 展開がマイナス |
| スコアなし | — | pace_yoso 未記入 |

ホバーでツールチップ表示: `展開適合指数: {値} / ペース予想: {H/M/S}`

## 8. DB テーブル構造

### T_PACEFIT_FACTOR_AGG（集計テーブル）

| カラム | 型 | 説明 |
|--------|-----|------|
| `factor_type` | VARCHAR(30) | ファクター種別 |
| `factor_value` | VARCHAR(40) | ファクター値 |
| `course_code` | CHAR(2) | 場コード（空=全体） |
| `tds_code` | CHAR(1) | 芝ダ（空=全体） |
| `dist_band` | VARCHAR(12) | 距離帯（空=全体） |
| `total_count` | INT | 集計件数 |
| `win_recovery` | DECIMAL(7,1) | 単勝回収率（円/馬） |
| `place_recovery` | DECIMAL(7,1) | 複勝回収率（円/馬） |

PRIMARY KEY: `(factor_type, factor_value, course_code, tds_code, dist_band)`

### T_PACEFIT_SCORE（スコアテーブル）

| カラム | 型 | 説明 |
|--------|-----|------|
| `course_code`〜`uma_num` | CHAR | レース・馬の複合PK |
| `pace_yoso` | CHAR(1) | H / M / S |
| `ten_juni` | TINYINT | テン指数順位 |
| `agari_juni` | TINYINT | 上がり指数順位 |
| `has_ten_gap` | TINYINT(1) | テンギャップフラグ |
| `has_agari_gap` | TINYINT(1) | 上がりギャップフラグ |
| `overall_score` | DECIMAL(7,1) | **出馬表表示用スコア** |
| `course_score` | DECIMAL(7,1) | コース補正スコア（非表示） |
| `score_ten_o` | DECIMAL(7,1) | テンスコア内訳 |
| `score_agari_o` | DECIMAL(7,1) | 上がりスコア内訳 |
| `score_gap` | DECIMAL(7,1) | ギャップボーナス内訳 |

## 9. ETL 実行手順

「データ取込」ページ → **「展開適合指数を更新する」** ボタン

内部では `sql/pacefit_index.sql` を以下の 4 パートに分けて実行する。

| Part | 処理 |
|------|------|
| 1 | テーブル定義（CREATE TABLE IF NOT EXISTS） |
| 2 | 全体ファクター集計（T_PACEFIT_FACTOR_AGG） |
| 3 | コース別ファクター集計（T_PACEFIT_FACTOR_AGG） |
| 4 | スコア計算・投入（T_PACEFIT_SCORE） |

Part2〜4 は `ON DUPLICATE KEY UPDATE` により冪等（再実行可能）。

## 10. 検討・除外したファクター

| ファクター | 除外理由 |
|-----------|---------|
| `ichi_index_juni`（位置指数順位） | M pace で分布が偏り単調性なし |
| `goal_juni`（ゴール順位） | ten/agari で十分、追加効果は限定的 |
| `ls_index_juni`（LS指数順位） | 9+帯が極端に少なく不安定 |
| `pace_index_juni`（ペース指数順位） | H pace の OOS が大幅ドロップ |
| `course_score`（コース補正） | 小セル過学習（OOS で大幅悪化） |
