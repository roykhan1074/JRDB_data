## 2. Conditions（絞り込み条件）

レース・出走条件でデータ母集団を絞り込む。**T_BAC** の項目が対象。

| 項目ID | ラベル | 型 | 送信パラメータ | DB列 | 備考 |
|---|---|---|---|---|---|
| `ymd_from` | 期間（開始） | テキスト | `ymd_from` | `b.ymd` | YYYYMMDD形式 |
| `ymd_to` | 期間（終了） | テキスト | `ymd_to` | `b.ymd` | YYYYMMDD形式 |
| `course` | 競馬場 | select | `course` | `b.course_code` | `00`=すべて |
| `tds` | 芝/ダート | select | `tds` | `b.tds_code` | `00`=すべて（後述の注意あり） |
| `distance_from` | 距離（下限） | select | `distance_from` | `b.distance` | 両方指定時のみ有効 |
| `distance_to` | 距離（上限） | select | `distance_to` | `b.distance` | 両方指定時のみ有効 |
| `class` | クラス | select | `class` | `b.class` | `00`=すべて（後述の注意あり） |

### ⚠️ デフォルト動作の注意点

| 条件 | デフォルト動作 | SQLで生成される句 |
|---|---|---|
| 芝/ダートを「すべて」のまま | **障害を自動除外** | `AND b.tds_code IN ('1','2')` |
| クラスを「すべて」のまま | **新馬（A1）を自動除外** | `AND b.class <> 'A1'` |

> 障害レース・新馬を含めたい場合は、明示的に「障害」「新馬」を選択する必要がある。

## 5. 出力項目定義

| 列名（画面） | APIキー | 計算式（SQL） | 型 |
|---|---|---|---|
| 集計キー（可変） | `agg_key_1`〜`agg_key_3` | 集計キー選択式 | 文字列 |
| 総数 | `total_heads` | `COUNT(*)` | 整数 |
| 1着 | `first_number` | `SUM(fin.f1)` | 整数 |
| 2着 | `second_number` | `SUM(fin.f2)` | 整数 |
| 3着 | `third_number` | `SUM(fin.f3)` | 整数 |
| 着外 | `also_ran` | `SUM(fin.fo)` | 整数 |
| 勝率% | `first_rate` | `ROUND(SUM(f1)/COUNT(*)*100, 1)` | 小数1桁 |
| 連対率% | `second_rate` | `ROUND((SUM(f1)+SUM(f2))/COUNT(*)*100, 1)` | 小数1桁 |
| 複勝率% | `third_rate` | `ROUND((SUM(f1)+SUM(f2)+SUM(f3))/COUNT(*)*100, 1)` | 小数1桁 |
| 単回収率 | `win_recovery_rate` | `ROUND(SUM(win_amt)/COUNT(*), 1)` | 小数1桁 |
| 複回収率 | `place_recovery_rate` | `ROUND(SUM(place_amt)/COUNT(*), 1)` | 小数1桁 |

### 色分けルール（単回収率・複回収率）

| 条件 | CSSクラス | 色 |
|---|---|---|
| ≥ 100 | `verygood` | 緑（濃） |
| ≥ 90 | `good` | 緑（淡） |
| < 60 | `bad` | 赤 |
| それ以外 | — | 無色 |

### サマリー行（合計）

集計キーが1つ以上あり、結果が2行以上の場合に末尾に表示。

| 項目 | 計算方法 |
|---|---|
| 総数・各着数 | Σ（各行の値） |
| 勝率・連対率・複勝率 | Σ着数 / 総数 × 100（直接集計） |
| 単回収率・複回収率 | Σ（行の回収率 × 行の総数） / 全体総数（**加重平均**） |

> サマリーの単複回収率は、各行の「すでにROUNDされた回収率」を使って加重平均するため、ROUNDによる微小な丸め誤差が生じる可能性がある。

## 7. SQLの構造

```sql
SELECT
  (集計キーSELECT式) AS agg_key_1,   -- 集計キー数に応じて増減
  COUNT(*)                             AS total_heads,
  SUM(fin.f1)                          AS first_number,
  SUM(fin.f2)                          AS second_number,
  SUM(fin.f3)                          AS third_number,
  SUM(fin.fo)                          AS also_ran,
  ROUND(SUM(fin.f1)/COUNT(*)*100,1)    AS first_rate,
  ROUND((SUM(fin.f1)+SUM(fin.f2))/COUNT(*)*100,1) AS second_rate,
  ROUND((SUM(fin.f1)+SUM(fin.f2)+SUM(fin.f3))/COUNT(*)*100,1) AS third_rate,
  ROUND(SUM(fin.win_amt)/COUNT(*),1)   AS win_recovery_rate,
  ROUND(SUM(fin.place_amt)/COUNT(*),1) AS place_recovery_rate

FROM t_bac b
INNER JOIN t_kyi k  ON (レースキー一致)
INNER JOIN (
  SELECT ...,
    CAST(TRIM(win)   AS UNSIGNED) AS win_amt,     -- 単勝払戻
    CAST(TRIM(place) AS UNSIGNED) AS place_amt,   -- 複勝払戻
    CAST(TRIM(order_of_finish) AS UNSIGNED) = 1   AS f1,
    CAST(TRIM(order_of_finish) AS UNSIGNED) = 2   AS f2,
    CAST(TRIM(order_of_finish) AS UNSIGNED) = 3   AS f3,
    CAST(TRIM(order_of_finish) AS UNSIGNED) >= 4  AS fo
  FROM t_sed
  WHERE ijou_kubun IN ('0','')       -- 正常完走のみ（取消・除外・失格を除く）
) fin ON (レースキー + 馬番一致)
LEFT JOIN t_cyb c  ON (レースキー + 馬番一致)  -- 追切・仕上指数はNULL許容
WHERE
  [Conditions句]
  [Targets句]
GROUP BY [集計キーGROUP BY式]
ORDER BY [集計キーORDER BY式]
LIMIT 2000
```

### JOINの関係

```
T_BAC（レース基本）
  ↓ INNER JOIN（レースキー）
T_KYI（出走表・指数）
  ↓ INNER JOIN（レースキー + 馬番）
T_SED（成績・払戻）← 異常区分フィルタあり
  ↓ LEFT JOIN（レースキー + 馬番）
T_CYB（調教分析）← 追切・仕上指数が未入力の馬はNULL
```

## 8. ⚠️ 確認が必要な設計上の疑問点

### 8-1. T_SED の `win`（単勝払戻）フィールドの格納仕様

現在の回収率計算式：

```sql
ROUND(SUM(CAST(TRIM(win) AS UNSIGNED)) / COUNT(*), 1) AS win_recovery_rate
```

この式が正しく機能するかは、**T_SED の `win` 列に何が入っているか** に依存する。

| パターン | T_SED.win の内容 | 計算結果 | 正否 |
|---|---|---|---|
| **A** | 1着馬の行のみ払戻額（例：560）、他は 0 | `560 / N頭` = 正しい単勝回収率 | ✅ 正しい |
| **B** | 全馬の行に同じ払戻額が入る（例：全行が560） | `560×N / N` = 560（払戻額そのまま） | ❌ 間違い |

→ **実データのSQLで確認**:

```sql
-- 同一レース内の win 列がどうなっているか確認
SELECT course_code, year_code, kai, day_code, race_num,
       umaban, order_of_finish, win, place
FROM T_SED
WHERE course_code='08' AND ymd='20260426'
ORDER BY race_num, CAST(TRIM(order_of_finish) AS UNSIGNED);
```

### 8-2. `place`（複勝払戻）フィールドの格納仕様

| パターン | T_SED.place の内容 | 計算結果 | 正否 |
|---|---|---|---|
| **A** | 3着以内の行のみ払戻額、他は 0 | `Σ払戻 / N頭` = 正しい複勝回収率 | ✅ 正しい |
| **B** | 全馬に同じ値が入る | ❌ 間違い |
| **C** | 払戻下限のみ入る（上限は別フィールド） | 実態に近い可能性あり | 要確認 |

### 8-3. `後3F順位`（`latter`）の集計キーと対応するTargetフィルタの不一致

| 区分 | 使用列 |
|---|---|
| Targetフィルタ（後3F順位） | `k.ato3f_juni`（T_KYI） |
| 集計キー「後3F順位」の SELECT/GROUP BY | `k.ato3f_juni`（T_KYI） |

→ 一致しているので問題なし。ただし「後3F順位」が**予想値**（出走前のJRDB予想）なのか**実績値**（レース後のJRDB確定値）なのかは要確認。T_KYI は出走前データなので予想値。

### 8-4. `前3F順位`（`first`）の画面表示名と内部名の確認

| 区分 | 画面ラベル | DB列 | 備考 |
|---|---|---|---|
| Targetフィルタ | 前3F順位 | `k.ten_index_juni` | テン指数順位（先行力） |
| 集計キー | 前3F順位 | `k.ten_index_juni` | 同上 |

→ 「テン指数順位」＝「前3F順位」として扱っている。JRDBのテン指数は前半3Fの速度指数であり、実際の前3F順位とは異なる概念の可能性がある。
