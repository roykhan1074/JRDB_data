## 4. API仕様

### POST /api/run

**リクエスト**

```json
{
  "dateFrom": "20260428",
  "dateTo":   "20260428",
  "prefixes": ["BAC", "KYI", "CYB", "SED", "UKC", "SRB"]
}
```

**レスポンス（SSE: Server-Sent Events）**

| イベントデータ | 型 | 説明 |
|---|---|---|
| `progress.current` | number | 現在処理中の日付インデックス |
| `progress.total` | number | 総日数 |
| `progress.date` | string | 現在処理中の日付（YYYYMMDD） |
| `message` | string | ログメッセージ（[完了]/[スキップ]/[エラー]等） |
| `done` | boolean | 全処理完了フラグ |
| `cancelled` | boolean | キャンセルされた場合 true |
| `error` | string | エラー発生時のメッセージ |

| クラス | 色 | 対象ログ |

## 6. ステータスバー

| 状態 | 背景色 | 条件 |
|---|---|---|
| running | 黄（#fff3cd） | 処理実行中 |
| done | 水色（#d1edff） | 正常完了 |
| error | 赤（#fde2e4） | エラー発生 |
| cancelled | オレンジ（#fdebd0） | キャンセル実行 |

---

## 7. 内部処理フロー

```
入力検証
  → POST /api/run（SSE接続）
    → 日付ごとにループ
      → downloadJRDB（zip取得 → text展開）
      → CONVERTERS[prefix]（txt → CSV変換）
      → loadCSVFile（CSV → MySQL UPSERT）
    → done イベント送信
```

- 既存ファイル（zip+txt）が両方存在する場合はダウンロードをスキップ
- `ON DUPLICATE KEY UPDATE` により再実行は冪等

## 8. EX指数更新

JRDBデータ取込とは独立した別処理。画面下部に専用セクションを設置。

### 概要

`sql/anaba_index.sql` を4パートに分割して順次実行し、EX指数（T_ANABA_SCORE）を再計算する。  
詳細は [EX指数仕様書](../分析レポート/穴馬指数_仕様書.md) を参照。

### UI

| 要素 | 説明 |
|---|---|
| 「EX指数を更新する」ボタン | `POST /api/anaba-etl` を呼び出す |
| SSEログエリア | Part1〜4の進捗をリアルタイム表示 |
| ステータスバー | running / done / error を色分け表示 |

### API: POST /api/anaba-etl

SSEで進捗を返す。リクエストボディなし。

| SSEイベント | 説明 |
|---|---|
| `message` | Part進捗ログ（例: `[Part3] ファクター集計 完了`） |
| `done: true` | 全パート完了 |
| `error: true` | エラー発生（処理中断） |

所要時間: Part4（指数計算）は約3〜5分かかるため、ハートビートメッセージを15秒ごとに送信。
