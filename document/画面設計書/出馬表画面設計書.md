## 3. レースヘッダー（race-header-card）

### 3-1. レースナビゲーション（rh-nav）

同日同場の全レースをボタン表示。現在のレースは `active` スタイル（濃紺背景）。  
データソース: `GET /api/races?date={ymd}&course={course_code}`

### 3-2. レース基本情報（rh-info）

| 表示要素 | データソース | 備考 |
|---|---|---|
| レース番号 | `race.race_num` | 整数表示 |
| レース名 | `race.race_name_9char` or `race.race_name` | 9文字略称優先 |
| クラスバッジ | `race.grade` + `race.class` | gradeが優先 |
| 発走時刻 | `race.start_time` | HHMM → H:MM |
| 芝/ダバッジ | `race.tds_code` | 1:芝 2:ダ 3:障害 |
| 左右 | `race.migihidari` | 1:右 2:左 |
| 距離 | `race.distance` + "m" | |
| 頭数 | `race.heads` + "頭" | |
| 一番人気信頼度 | calcFavoriteReliability() で計算 | 後述 |

#### クラスバッジ対応表

| race.grade | 表示 |
|---|---|
| `1` | G1（黄バッジ） |
| `2` | G2 |
| `3` | G3 |
| `A` | L |

| race.class | 表示 |
|---|---|
| `A1` | 新馬 |
| `A3` | 未勝利 |
| `05` | 1勝 |
| `10` | 2勝 |
| `16` | 3勝 |
| `OP` | OP |

### 3-3. 一番人気信頼度（fav-rel-meta）

`kijun_ninki = 1` の馬（1番人気）について、JRDB指数群がその評価を支持しているかをS〜Eで表示。  
オッズは**入力に使わない**（市場評価から独立した指標）。  
ホバーでスコア内訳ツールチップを表示。フロントエンド `calcFavoriteReliability()` で計算。

| グレード | スコア | 意味 |
|---|---|---|
| S | ≥ 70 | 全指数が強く支持 |
| A | 55〜69 | 信頼できる本命 |
| B | 25〜54 | 標準（全体1番人気平均） |
| C | 0〜24 | 平均より下 |
| D | −20〜−1 | 支持が薄い |
| E | < −20 | 偽本命の可能性 |

配色：sp-a〜sp-e に準拠（S=金 / A=赤 / B=緑 / C〜D=グレー / E=薄グレー）

**スコアリング内訳**

| ファクター | フィールド | 最高 | 最低 |
|---|---|---|---|
| IDM優位性 | `idm` のレース内順位・差分 | +40 | −20 |
| 情報指数一致 | `joho_index` のレース内順位 | +20 | −15 |
| 展開有利性 | `goal_juni` のレース内順位 | +15 | −10 |
| 調教矢印 | `chokyo_yajirushi` | +15 | −20 |
| 追切指数 | `oi_index` のレース内順位 | +8 | −5 |
| 仕上指数 | `shiage_index` のレース内順位 | +8 | −5 |
| ペース適性 | `ten_index_juni`+`agari_index_juni` | +10 | −6 |

`joho_index = −1`（非開示）の場合は当該ファクターを中立（0点）として除外。

### 4-4. 騎×厩グレード（現在非表示）

列は `width:0` で非表示中。データ自体はAPIから返却されており、再表示時は col幅を 48px に戻すだけで有効化できる。

`T_COMBO_RECOVERY.place_recovery`（騎手コード×調教師コードの組み合わせ複勝回収率）をA〜Eで評価。

| グレード | 複勝回収率 | セル背景 |
|---|---|---|
| A | 100%以上 | ピンク（#fbcfe8） |
| B | 90%以上 | 緑（#bbf7d0） |
| C | 80%以上 | なし |
| D | 65〜79% | なし |
| E | 65%未満 | グレー（#e5e7eb） |

### 4-5. 厩舎指数の表示ルール

| 条件 | 数値スタイル | セル背景 |
|---|---|---|
| `kyusha_index >= 10` | 赤文字太字（stable-good） | なし |
| `kyusha_index < 0` | 紫文字（stable-bad） | なし |
| `kijun_odds >= 15` かつ `kyusha_index >= 0` かつ `kyusha_anaba_place_rr >= 100%` | そのまま | ピンク（stable-pink） |
| `kijun_odds >= 15` かつ `kyusha_index >= 0`（上記以外） | そのまま | 黄色（stable-yellow `#fef9c3`） |
| その他 | デフォルト | なし |

ピンク条件が成立する場合、厩舎名セルのホバーで「穴信頼度(複回収): XX%（N件）」ツールチップを表示。

### 4-6. EX指数（Explosion Index）

バックテスト（約17万件）で検証した変曲点（0・20・50・100）に基づく絶対値配色。  
**全馬にスコアを表示する。ただしオッズ10倍未満の馬はセル背景色を付けない。**

| スコア | セル背景 | テキスト | 単勝回収率実績（穴馬対象） |
|---|---|---|---|
| 100以上 | 薄赤 `#f87171` | 黒・太字 | 374% |
| 50〜99 | ピンク `#fbcfe8` | 黒 | 90〜177% |
| 20〜49 | 黄色 `#fef9c3` | 黒 | 83〜87% |
| 0〜19 | なし（白） | 黒 | 71〜73% |
| 0未満 | グレー `#e5e7eb` | グレー文字 | 40〜65% |

> セル背景色はオッズ10倍以上の馬（穴馬）のみ適用。10倍未満の馬はスコア数値のみ表示。

ホバーでEX指数内訳ツールチップ（13ファクターの寄与値）を表示。

**重点買いハイライト（chukiten）**

以下を同時に満たす馬の行を薄緑背景（#f0fdf4）で強調表示。

- `kijun_odds >= 10` かつ `kijun_odds < 50`（中穴）
- 調教スコア `>= 70`

## 5. 調教SP

詳細は [調教SP仕様書](../分析レポート/調教SP_仕様書.md) を参照。

### 算出概要（calcChokyoSP）

同レース内の追切指数・仕上指数のレース内順位ボーナス（最大+6）に、z-scoreネガティブ調整・調教矢印ペナルティ・放牧先ランクペナルティを加算したスコアをA〜Eに変換。

| グレード | スコア | セル背景 |
|---|---|---|
| A | 4以上 | ピンク（#fbcfe8） |
| B | 2〜3 | 緑（#bbf7d0） |
| C | −1〜1 | なし |
| D | −4〜−2 | なし |
| E | −5以下 | グレー（#e5e7eb） |

計算はフロントエンド（JavaScript）で実施。

## 6. API仕様

### GET /api/races/:raceKey（レース情報）

T_BAC からレース基本情報を返す。

### GET /api/races?date={ymd}&course={course_code}（同日同場レース一覧）

レースナビゲーションボタン生成に使用。

### GET /api/races/:raceKey/entries（出走馬一覧）

**レスポンス：** `{ source: "entries", rows: [...] }`

JOIN: T_KYI LEFT JOIN T_CYB LEFT JOIN T_COMBO_RECOVERY LEFT JOIN T_KYUSHA_FACTOR_AGG(subquery) LEFT JOIN T_ANABA_SCORE LEFT JOIN T_SED
