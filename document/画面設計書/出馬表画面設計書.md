## 4. 出走馬テーブル

テーブル最大幅：`1400px`

### 4-1. ヘッダー構造

2行構成。親行に全カラムを配置し、「騎×厩」のみ親行で colspan=1・rowspan=2 の単独ヘッダー。

### 4-2. カラム一覧

| 列 | DBカラム | 表示内容 | 備考 |
|---|---|---|---|
| 枠 | `waku_num` | 1〜8の枠番 | 枠色背景（waku-1〜waku-8） |
| 馬番 | `uma_num` | 馬番 | |
| 馬名 | `uma_name` | 馬名 | |
| B | `blinker` | B | 1=初装着/2=再装着→青太字 |
| 脚質 | `kyakushitsu` | 逃/先/差/追 | 1:逃げ 2:先行 3:差し 4:追込 |
| オッズ | `kijun_odds` | 基準単勝オッズ | 1位=ピンク背景、2〜3位=緑背景 |
| 騎手 | `kishu_name` | 騎手名 | |
| 厩舎 | `trainer_name` | 調教師名 | |
| 騎×厩 | `combo_place_rr` / `combo_n` | A〜E グレード | 複勝回収率で評価。nはホバー表示 |
| 情報印 | `joho_index` の Dense Rank | ◎○▲注△▽ | `joho_index=-1` のとき非表示・グレー背景 |
| 情報指数 | `joho_index` | 数値 | |
| 展開印 | `goal_juni` の Dense Rank | ◎○▲注△▽ | |
| 展開指数 | `goal_juni` | 数値 | |
| IDM印 | `idm` の Dense Rank | ◎○▲注△▽ | IDM3位との差が5pt未満→グレー背景 |
| IDM指数 | `idm` | 数値 | |
| 前3F | `ten_index_juni` | 順位 | 1位=ピンク、2〜3位=緑 |
| 後3F | `agari_index_juni` | 順位 | 1位=ピンク、2〜3位=緑 |
| 厩舎指数 | `kyusha_index` | 数値 | ≥10=赤、＜0=紫、基準オッズ≥15かつ≥0=黄背景 |
| 調教SP | 独自計算（後述） | A〜E グレード | |
| 着順 | `result_order`（T_SED） | 着順数字 | 1着=赤、2着=青、3着=緑。取消/除外はグレー |
| 単勝 | `result_win`（T_SED） | 払戻金額 | 1着馬のみ表示 |
| 複勝 | `result_place`（T_SED） | 払戻金額 | 1〜3着馬のみ表示 |

### 4-3. 印の順位→記号マッピング

| 順位 | 記号 | 色 |
|---|---|---|
| 1 | ◎ | 赤 |
| 2 | ○ | 緑 |
| 3 | ▲ | 緑 |
| 4 | 注 | 黒 |
| 5 | △ | 黒 |
| 6 | ▽ | 黒 |
| 7以上 | — | — |

印はレース内の Dense Rank（同値同順位）で計算。

### 4-4. 騎×厩グレード（複勝回収率）

T_COMBO_RECOVERY の `place_recovery`（騎手コード×調教師コードの組み合わせ回収率）をA〜Eで評価。

| グレード | 複勝回収率 | 表示スタイル |
|---|---|---|
| A | 100%以上 | 赤背景バッジ（rr-fire） |
| B | 90%以上 | ピンク文字（rr-90up） |
| C | 80%以上 | オレンジ文字（rr-80up） |
| D | 65〜79% | デフォルト（黒文字） |
| E | 65%未満 | グレー文字（rr-low） |

サンプル数（n）はセルにカーソルを当てるとツールチップで表示。

### 4-5. 着順・払戻の表示ルール

| 状態 | 着順表示 | 単勝表示 | 複勝表示 |
|---|---|---|---|
| 1着 | 赤太字 "1" | 払戻金額 | 払戻金額 |
| 2着 | 青太字 "2" | — | 払戻金額 |
| 3着 | 緑太字 "3" | — | 払戻金額 |
| 4着以下 | 黒 | — | — |
| 取消 | グレー "取消" | — | — |
| 除外 | グレー "除外" | — | — |
| 未格納（レース前） | — | — | — |

## 5. 調教SP

詳細は [調教SP仕様書](../分析レポート/調教SP_仕様書.md) を参照。

### 算出概要

同レース内の追切指数・仕上指数のレース内順位ボーナス（最大+6）に、z-scoreネガティブ調整・調教矢印ペナルティ・放牧先ランクペナルティを加算したスコアをA〜Eに変換。

| グレード | スコア | 単回収率（バックテスト） |
|---|---|---|
| A | 4以上 | 88% |
| B | 2〜3 | 79% |
| C | −1〜1 | 73% |
| D | −4〜−2 | 65% |
| E | −5以下 | 52% |

計算はフロントエンド（JavaScript）で全馬データを受け取った後に実施。バックエンド変更不要。

## 6. API仕様

### GET /api/races/:raceKey（レース情報）

T_BAC からレース基本情報を返す。

```json
{
  "course_code": "08", "year_code": "26", "kai": "3",
  "day_code": "2",     "race_num": "10",
  "ymd": "20260426",   "race_name": "センテニアル・ＰＳ",
  "race_name_9char": "センテニアルＰＳ",
  "distance": "1800",  "tds_code": "1",
  "grade": "5",        "class": "16",
  "heads": "18",       "start_time": "1450",
  "migihidari": "1",   "naigai": "1"
}
```

### GET /api/races/:raceKey/entries（出走馬一覧）

**レスポンス：** `{ source: "entries", rows: [...] }`

T_KYI LEFT JOIN T_CYB LEFT JOIN T_COMBO_RECOVERY LEFT JOIN T_SED
