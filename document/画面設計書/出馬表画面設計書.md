## 3. 統合レースヘッダー（race-header-card）

ヘッダーは左エリアと右エリアで構成される1枚カード。

### 3-1. 左エリア（race-header-left）

**レースナビゲーション（rh-nav）**

同日同場の全レースをボタン表示。現在のレースは `active` スタイル（濃紺背景）。  
データソース: `GET /api/races?date={ymd}&course={course_code}`

**レース基本情報（rh-info）**

| 表示要素 | データソース | 備考 |
|---|---|---|
| レース番号 | `race.race_num` | 整数表示 |
| レース名 | `race.race_name_9char` or `race.race_name` | 9文字略称優先 |
| クラスバッジ | `race.grade` + `race.class` | gradeが優先 |
| 発走時刻 | `race.start_time` | HHMM → H:MM |
| 芝/ダバッジ | `race.tds_code` | 1:芝 2:ダ 3:障害 |
| 左右 | `race.migihidari` | 1:右 2:左 |
| 距離 | `race.distance` + "m" | |
| 頭数 | `race.heads` + "頭" | |

### 3-2. 右エリア：本命信頼度 + コース別ファクターパネル（race-header-right）

**本命信頼度（fav-rel-panel）**

`kijun_ninki = 1` の馬（1番人気）について、JRDB指数群がその評価を支持しているかをA〜Eで表示。  
オッズは**入力に使わない**（市場評価から独立した指標）。  
ホバーでスコア内訳ツールチップを表示。

| グレード | スコア | 実績勝率 | 意味 |
|---|---|---|---|
| S | ≥ 70 | 43.4% | 全指数が強く支持 ＝「本物の本命・迷わず買える」 |
| A | 55〜69 | 36.0% | 信頼できる本命 |
| B | 25〜54 | 30.3% | 標準（全体の1番人気平均31.4%に一致） |
| C | 0〜24 | 24.3% | 平均より下 ＝ 要注意 |
| D | −20〜−1 | 19.4% | 支持が薄い ＝ 危険信号 |
| E | < −20 | 13.0% | 複数の赤フラグ ＝「偽本命の可能性大」 |

配色：S=金(#ca8a04) / A=赤 / B=緑 / C=グレー / D=濃グレー / E=薄グレー（sp-a〜sp-e に準拠）

※ 実績は2020〜2024年の約20,900レースによるバックテスト結果

**スコアリング内訳（全てJRDB指数のレース内相対ランクで評価）**

| ファクター | フィールド | 最高 | 最低 | 判定軸 |
|---|---|---|---|---|
| IDM優位性 | `idm` のレース内順位・差分 | +40 | −20 | 1位かつ差≥5で最高点 |
| 情報指数一致 | `joho_index` のレース内順位 | +20 | −15 | 1位=一致、6位以下=乖離 |
| 展開有利性 | `goal_juni` のレース内順位 | +15 | −10 | 値小＝有利（昇順） |
| 調教矢印 | `chokyo_yajirushi` | +15 | −20 | ↑↑=+15、↓↓=−20 |
| 追切指数 | `oi_index` のレース内順位 | +8 | −5 | 1位=+8 |
| 仕上指数 | `shiage_index` のレース内順位 | +8 | −5 | 1位=+8 |
| ペース適性 | `ten_index_juni`+`agari_index_juni` | +10 | −6 | 1位×2=+10 |

`joho_index = −1`（非開示）の場合は当該ファクターを中立（0点）として除外。

### 3-3. 右エリア：コース別ファクターパネル（race-header-right）

`race.distance` と `race.tds_code` の両方が存在する場合のみ表示。  
データソース: `GET /api/course-analysis?course_code={}&tds_code={}&distance={}`

IDM印・前3F（テン）・後3F（上がり）について、レース内順位帯（1位 / 2〜3位 / 4〜6位）ごとに複勝回収率をA〜Eグレードで表示。

| グレード | 複勝回収率 | 表示スタイル |
|---|---|---|
| A | 100%以上 | 赤文字太字（rr-fire） |
| B | 90%以上 | ピンク文字（rr-90up） |
| C | 80%以上 | オレンジ文字（rr-80up） |
| D | 65〜79% | デフォルト（黒文字）（cf-d） |
| E | 65%未満 | グレー文字（rr-low） |
| — | データなし | グレー点線 |

#### クラスバッジ対応表

| race.grade | 表示 |
|---|---|
| `1` | G1（黄バッジ） |
| `2` | G2 |
| `3` | G3 |
| `A` | L |

| race.class | 表示 |
|---|---|
| `A1` | 新馬 |
| `A3` | 未勝利 |
| `05` | 1勝 |
| `10` | 2勝 |
| `16` | 3勝 |
| `OP` | OP |

### 4-3. 印の順位→記号マッピング

| 順位 | 記号 | 色 |

印はレース内の Dense Rank（同値同順位）で計算。

### 4-4. 騎×厩グレード（複勝回収率）

`T_COMBO_RECOVERY.place_recovery`（騎手コード×調教師コードの組み合わせ複勝回収率）をA〜Eで評価。

| グレード | 複勝回収率 | セル背景 | 文字スタイル |
|---|---|---|---|
| A | 100%以上 | ピンク（#fbcfe8） | 赤文字太字（sp-a） |
| B | 90%以上 | 緑（#bbf7d0） | 緑文字（sp-b） |
| C | 80%以上 | なし | 黒文字（sp-c） |
| D | 65〜79% | なし | 黒文字（sp-d） |
| E | 65%未満 | グレー（#e5e7eb） | グレー文字（sp-e） |

サンプル数（n）・単勝回収率はセルにカーソルを当てるとツールチップで表示。

### 4-5. 厩舎指数の表示ルール

| 条件 | 数値スタイル | セル背景 |
|---|---|---|
| `kyusha_index >= 10` | 赤文字太字（stable-good） | なし（デフォルト） |
| `kyusha_index < 0` | 紫文字（stable-bad） | なし |
| `kijun_odds >= 15` かつ `kyusha_index >= 0` かつ `kyusha_anaba_place_rr >= 100%` | そのまま | ピンク背景（stable-pink） |
| `kijun_odds >= 15` かつ `kyusha_index >= 0`（上記以外） | そのまま | 黄色背景（stable-yellow） |
| その他 | デフォルト | なし |

ピンク条件が成立する場合、厩舎名セルのホバーで「穴信頼度(複回収): XX%（N件）」ツールチップを表示。

### 4-6. 穴馬指数（anaba）

`kijun_odds >= 10.0` の馬のみスコアを表示。それ以外は `—` グレー表示。

**スコア配色**（穴馬対象馬内での Dense Rank による）:

| 順位 | 色 | 太さ | セル背景 |
|---|---|---|---|
| 1位 | 赤（#dc2626） | 700 | ピンク（#fbcfe8） |
| 2〜3位 | 緑（#15803d） | 600 | 緑（#bbf7d0） |
| 4位以下 | 黒 | 400 | なし |
| マイナス値 | グレー（#9ca3af） | — | なし |

ホバーで穴馬指数内訳ツールチップ（13ファクターの寄与値）を表示：

> テン / 上がり / 位置 / ゴール / 展開複合 / IDM / 激走 / 万券 / 調教 / 脚質 / 上昇度 / 適性 / 血統

**重点買いハイライト（chukiten）**

以下を同時に満たす馬の行を薄緑背景（#f0fdf4）・左端3px緑ラインで強調表示。

- `kijun_odds >= 10` かつ `kijun_odds < 50`（中穴：10〜49倍）
- 調教スコア `>= 70`（calcTrainingScore による）

## 5. 調教SP

詳細は [調教SP仕様書](../分析レポート/調教SP_仕様書.md) を参照。

### 算出概要（calcChokyoSP）

同レース内の追切指数・仕上指数のレース内順位ボーナス（最大+6）に、z-scoreネガティブ調整・調教矢印ペナルティ・放牧先ランクペナルティを加算したスコアをA〜Eに変換。

| グレード | スコア | セル背景 |
|---|---|---|
| A | 4以上 | ピンク（#fbcfe8） |
| B | 2〜3 | 緑（#bbf7d0） |
| C | −1〜1 | なし |
| D | −4〜−2 | なし |
| E | −5以下 | グレー（#e5e7eb） |

計算はフロントエンド（JavaScript）で全馬データを受け取った後に実施。バックエンド変更不要。

## 6. API仕様

### GET /api/races/:raceKey（レース情報）

T_BAC からレース基本情報を返す。

```json
{
  "course_code": "08", "year_code": "26", "kai": "3",
  "day_code": "2",     "race_num": "10",
  "ymd": "20260426",   "race_name": "センテニアル・ＰＳ",
  "race_name_9char": "センテニアルＰＳ",
  "distance": "1800",  "tds_code": "1",
  "grade": "5",        "class": "16",
  "heads": "18",       "start_time": "1450",
  "migihidari": "1",   "naigai": "1"
}
```

### GET /api/races?date={ymd}&course={course_code}（同日同場レース一覧）

レースナビゲーションボタン生成に使用。

### GET /api/course-analysis?course_code={}&tds_code={}&distance={}（コース別ファクター）

コース別ファクターパネルに使用。`distance` と `tds_code` が両方ある場合のみ呼び出す。

### GET /api/races/:raceKey/entries（出走馬一覧）

**レスポンス：** `{ source: "entries", rows: [...] }`

JOIN: T_KYI LEFT JOIN T_CYB LEFT JOIN T_COMBO_RECOVERY LEFT JOIN T_KYUSHA_FACTOR_AGG LEFT JOIN T_ANABA_SCORE LEFT JOIN T_SED
