## 3. 検索フォーム

| ID | 種別 | ラベル | 仕様 |
|---|---|---|---|
| `filter-input` | テキスト入力 | 騎手名 | 入力内容でプルダウンをリアルタイム絞り込み。部分一致 |
| `kishu-sel` | `<select>` | (騎手選択) | 全騎手307件を五十音順で表示。変更時に自動で能力カードを描画 |
| `year-sel` | `<select>` | 年度 | 2020〜当年を選択肢として表示。「最新」選択時はDBの最新年度を使用 |
| `btn-search` | ボタン | 表示 | プルダウン選択中の騎手を指定年度で表示する（selectの`change`でも同様に動作） |

### 初期状態
- ページ読み込み時に `GET /api/jockeys` を呼び出し、全騎手をプルダウンに展開する
- URLパラメータ `?name=<騎手名>&year=<年度>` が付いている場合は該当騎手を選択状態にして能力カードを描画する

## 4. 能力カード

### 4.1 カードヘッダー

| 表示要素 | 内容 | データソース |
|---|---|---|
| 騎手コード | 黄色大文字で左端に表示（例：`10512`） | `T_KISHU_FACTOR_AGG.kishu_code` |
| 騎手名 | 太字（例：川田将雅） | URLパラメータ / プルダウン選択値 |
| 年度タブ | 当該騎手のデータが存在する年度をボタンで列挙。クリックで切り替え | `T_KISHU_RACE_LOG` の `SUBSTRING(ymd,1,4)` の DISTINCT |
| 総合成績 | 出走数・勝率・複勝率・単勝回収率・複勝回収率 | `T_KISHU_RACE_LOG` を直接集計（異常区分 = '0' or '' のみ） |

---

### 4.2 左パネル：基本スペック

**集計指標：複勝回収率**（`place_payout_sum / total_count`）を使用。

#### スペック項目定義

| 項目名 | factor_type | factor_value（複数時は合算） | 最低出走数 |
|---|---|---|---|
| 芝 | `tds` | `1` | 20 |
| ダート | `tds` | `2` | 20 |
| 短距離 | `distance` | `~1200` + `1201~1400` | 15 |
| マイル | `distance` | `1401~1600` | 15 |
| 中距離 | `distance` | `1601~2000` | 15 |
| 長距離 | `distance` | `2001~2400` + `2401~` | 10 |
| 先行力 | `ten_rank` | `1`（テン指数1位） | 10 |
| 末脚力 | `agari_rank` | `1`（上がり指数1位） | 10 |

> 複数バケットの合算：`place_recovery = Σ(place_payout_sum) / Σ(total_count)`

#### グレード定義

| グレード | 複勝回収率 | 背景色 | 文字色 | 意味 |
|---|---|---|---|---|
| **S** | ≥ 95 | `#FFD700`（金） | `#1a1a2e` | 突出した強み |
| **A** | ≥ 85 | `#E02020`（赤） | 白 | 高水準 |
| **B** | ≥ 75 | `#2255CC`（青） | 白 | 平均以上 |
| **C** | ≥ 65 | `#2A8C2A`（緑） | 白 | 平均水準 |
| **D** | < 65 | `#888`（灰） | 白 | 平均以下 |
| **?** | 最低出走数未満 | `#CCC` | `#999` | データ不足 |

#### バー表示

```
バー幅(%) = max(2, min(100, (複勝回収率 − 50) / 70 × 100))
```
- 50回収 → 0%、120回収 → 100% にマッピング
- グレードに対応した色のグラデーションで描画

## 5. API仕様

### 5.1 騎手一覧取得

```
GET /api/jockeys?q=<名前>
```

| パラメータ | 必須 | 説明 |
|---|---|---|
| `q` | 任意 | 騎手名の部分一致フィルタ。省略時は全件（最大500件）を返す |

**レスポンス（JSON配列）**

```json
[
  { "kishu_name": "川田将雅", "kishu_code": "10512" },
  { "kishu_name": "横山武史", "kishu_code": "05339" }
]
```

**データソース**：`T_KISHU_FACTOR_AGG`（`factor_type='tds'` でユニーク）  
**ソート**：`kishu_name` 昇順  
**上限**：500件

---

## 6. データ定義

### 6.1 T_KISHU_RACE_LOG（騎手成績ログ）

| カラム | 型 | 説明 |
|---|---|---|
| course_code | CHAR(2) | 場コード（PK） |
| year_code | CHAR(2) | 年（PK） |
| kai | CHAR(1) | 回（PK） |
| day_code | CHAR(1) | 日（PK） |
| race_num | CHAR(2) | R（PK） |
| uma_num | CHAR(2) | 馬番（PK） |
| ymd | CHAR(8) | 年月日 YYYYMMDD |
| kishu_name | VARCHAR(12) | 騎手名 |
| kishu_code | CHAR(5) | 騎手コード |
| trainer_name | VARCHAR(12) | 調教師名 |
| umanushi_name | VARCHAR(40) | 馬主名 |
| distance | CHAR(4) | 距離（m） |
| tds_code | CHAR(1) | 芝ダ障害（1/2/3） |
| class | CHAR(2) | 条件クラス |
| heads | CHAR(2) | 頭数 |
| kijun_odds | CHAR(5) | 基準単勝オッズ |
| ten_index_juni | CHAR(2) | テン指数順位 |
| agari_index_juni | CHAR(2) | 上がり指数順位 |

| place_payout | INT | 複勝払戻（円/100円）0=非対象 |

### 6.2 T_KISHU_FACTOR_AGG（ファクター別集計）

| カラム | 型 | 説明 |

### 6.3 factor_type / factor_value 対応表

| factor_type | factor_value | 説明 |

| `trainer` | 調教師名（全角） | 調教師ごと |
| `umanushi` | 馬主名（全角） | 馬主ごと |

## 7. ETL（データ更新）

データの最新化は `sql/kishu_analysis.sql` を実行する。冪等処理（ON DUPLICATE KEY UPDATE）のため毎日再実行可能。

```bash
# 全量更新（初回 or 月次）
mysql -u<user> -p<pass> <dbname> < sql/kishu_analysis.sql

# 日次運用想定の実行順
# Part2: T_KISHU_RACE_LOG 投入  （新規レース分が追記される）
# Part3: T_KISHU_FACTOR_AGG 集計（全件再集計）
```

## 8. 注意事項・制約

| 項目 | 内容 |
|---|---|
| グレード基準は固定値 | 複勝回収率の絶対値で評価するため、リーディング上位騎手はほぼ全条件でB以上になることがある |
| サンプル数下限 | 各スペック・バッジには最低出走数の条件あり。少数サンプルは `?` グレードまたはバッジ非表示 |
| 馬主名・調教師名は長い場合がある | 馬主名は10文字で切り捨て表示（`.slice(0, 10)`） |
| 障害レースを含む | `tds_code='3'` の障害レースも集計対象。基本スペックには障害専用スペックなし |
| データ範囲 | `T_KISHU_RACE_LOG` の範囲：2020-01-05〜現在 |
