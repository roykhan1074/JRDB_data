## ⛔ UIカラー規約（最重要・何度も間違えている）

出馬表（entries.html）で A〜E グレードを表示するとき、色は必ず以下の規約に従う。
**勝手に変えない。** 既存の `sp-a`〜`sp-e` クラスがこの規約の正典。

| グレード | 背景色 | 文字色 | クラス参照 |
|---|---|---|---|
| S | `#ca8a04` | `#fff` | （特別枠） |
| A | `#dc2626` | `#fff` | `sp-a` 赤 |
| B | `#15803d` | `#fff` | `sp-b` 緑 |
| C | `#6b7280` | `#fff` | `sp-c` 系 |
| D | `#374151` | `#fff` | `sp-d` 系 |
| E | `#e5e7eb` | `#6b7280` | `sp-e` 薄グレー |

セル背景を使う場合:
- A → `#fbcfe8`（ピンク）
- B → `#bbf7d0`（薄緑）
- E → `#e5e7eb`（薄グレー）
- C/D → なし（白）

**よくやるミスの例:**
- A=緑、E=赤 にする（逆！）
- B を青に変える（禁止）
- アンバー・オレンジを C/D に使う（禁止）
- 新しいグレード体系を作るとき、A〜E の色を既存規約から変える（禁止）

## テーブル設計規約

- `table-layout: fixed` を必ず使う
- テーブル本体は `width: auto`（`width:100%` 禁止）。col の合計幅がそのままテーブル幅になる。`width:100%` にすると列を隠したとき他列に幅が分配されてレイアウトが崩れる
- すべての列を `<colgroup><col style="width:Xpx">` で明示する（幅未指定列の禁止）
- スクロールは `table-wrap` の `overflow-x: auto` で対応する
- `white-space: nowrap` と `overflow: hidden` をセルに適用する
- ヘッダーは `rowspan` / `colspan` で構造化する

## ⛔ UI変更時の必須チェック（何度もレイアウト崩れを起こしている）

**私（Claude）はブラウザレンダリングを目視確認できない。**
幅・高さ・折り返し・視覚的ずれは事前に保証できない。
できる検証は「列数の整合」のみ。それ以外はユーザーに確認を求める。

**テーブルに関わる変更をしたら、完了報告の末尾に必ず「レイアウトをご確認ください」と書くこと。省略禁止。**

### 1. 列数の整合チェック（私が実行できる唯一の機械的検証）
`col数 = 実効th数（colspan展開後）= td数` が一致しているか数える。
一致しない場合は原因を突き止めてから報告する。
※これで検出できるのは「列の数がずれている」ケースのみ。幅・高さの問題は検出不可。

### 3. 列の非表示方法（唯一の正しい方法）
```
<col style="width:0">                                          ← colで幅を0に
<th style="width:0;padding:0;border:0;overflow:hidden;font-size:0">  ← th
<td style="width:0;padding:0;border:0;overflow:hidden;font-size:0">  ← td
```
- `display:none` を `<th>` に使うと **colspan のカウントがずれる → 禁止**
- `visibility:collapse` はブラウザ互換性が低い → **禁止**
